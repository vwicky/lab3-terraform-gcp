# Хмарні технології - Лабораторна робота №3

**Виконав:** Зозуля Віктор (Варіант 6)
**Провайдер:** Google Cloud Platform (GCP)

## Передумови (Prerequisites)
1. Встановлений Terraform.
2. Встановлений та налаштований Google Cloud SDK (gcloud CLI).
3. Створений Service Account з правами Compute Admin та Storage Admin.
4. Створений GCS Bucket для зберігання віддаленого стану.

## Інструкція з використання (Terraform Core Workflow)
- `terraform init` — ініціалізація проєкту, налаштування backend (бакету) та завантаження плагінів провайдера GCP.
- `terraform plan` — планування: перевірка того, які саме ресурси будуть створені або змінені.
- `terraform apply` — розгортання: застосування конфігурації та створення інфраструктури у хмарі.
- `terraform destroy` — очищення: кероване видалення всіх створених ресурсів для уникнення зайвих витрат.
